Question: Elaborate on a few (3-5) different security issues or concerns, from your own experience, related to web application security and relate them to the OWASP top ten.

Answer: I think one of the most common security concerns are "Sensitive data exposure" and its also one concern of mine. I wouldn't want my data to be exposed for everyone to access. Attackers may steal or use the data in order to perform credit card fraud and identity theft. It hasn't happend to me, yet. Sensitive data exposure is on the OWASP top 10 which is used for developers in order to minimize the risks.

Another concern of mine is broken authentication. If the application functions related to authentication is incorrectly implemented, attackers can gain access to the password, session tokens etc. I wouldn't want my account to be hijacked since the account may contain messages, images, memories etc I woulnd't want to lose. The attackers can also post and use the accounts and pretend to be "me".

Another concern is injection which can lead to dataloss, corruption, denial of access. Attackers can also take complete control over the application due to injection. Imagine a business gets attacked and an injection is made on the database, this will affect all the users/customers as well. This is one of the most known security issues on OWASP.

Question: Summarize by providing your own guidelines to a new web programmer on the topic “The essential guidelines to follow to create secure web applications”.

Answer: Guidelines to create secure web applications:

1. Keep yourself updated and follow web applications security techniques.

The best way to stay safe against hacker is to be aware of popular and new insecurities.

2. Always have a back-up for your data.

If you've got a security breach and you need to restore your data, it would be catastrophic if you havent backed up your data and you need to start all over again.

3. Ask professionals to "hack your web application".

4. Scan your website often for vulnerablities

5. Use a web application security platforms

6. Strong password policy and encryption for login

Question: What is your TIL for this course section?

Answer: This course section, I've learned about the common security risks in a web application, how the attackers are using them and how to prevent it. I've also learned about OWASP top 10 security risks.
